DASHBOARD DESCRIPTION

  • This dashboard showcases NLab's generosity to its customers‚Äîoffering discounts or gifts as a demonstration of goodwill.
  • The data sources include Shopify for each e-shop and the Amazon Selling Partners database. The modeling was performed using dbt tools and BigQuery.
  • Data updates occur every morning at 8 AM.

NAVIGATION DESCRIPTION

The dashboard comprises five standalone dashboards, each dedicated to a specific e-shop: Nutrimuscle, Nutrielement, NLab Japan, NLab B2B, and Amazon Europe.

Each dashboard highlights the following metrics:

  • Total orders
  • Total sales
  • Total generosity (commercial and industrial)
  • UPT (Units Per Transaction)
  • AOV (Average Order Value)
  • Percentage of new customers within a selected period
  • A benchmark section comparing two periods
  • A global view since the brand‚Äôs launch, illustrating the contribution of each category and subcategory to generosity

KPI Definitions

Total Orders

Represents the unique count of order IDs recorded in each e-shop, excluding DOTA orders.

  • DOTA Order Definition: An order is classified as DOTA if it includes a discount code containing the term "DOTA".

üìå Formula in the Data Warehouse:

sql
Orders = COUNT(DISTINCT shopify__order__id)

Gross Sales (VAT Included)

Represents the total sum of the product prices in each order, including any applied discounts.

üìå Formula in the Data Warehouse:

sql
Gross Sales = SUM(shopify__order__line_item__price)

Total Sales

Represents the total revenue, including product price, shipping fees, and taxes, while deducting total generosity (discounts & incentives).

üìå Formula in the Data Warehouse:

sql
Total Sales = SUM(shopify__order__line_item__price + shopify__order__tax__value + shopify__order__shipping__value - shopify_total_generosity)

Total Generosity

Represents the total sum of all applied discounts per order, adjusted for industrial generosity deductions

‚ö†Ô∏è Note:

Starting January 1, 2025, commercial generosity will be calculated using a new methodology. The new formula excludes discounts from GLADY codes, customer care codes (e.g., transport, logistics, stock), shipping fees, and DOTA-related incentives.

üìå Formula in the Data Warehouse:

sql
Total Generosity = SUM(total_acquisition + total_loyalty + total_customer_care + total_partners_staff)

UPT (Units Per Transaction)

Represents the total number of purchased items (excluding gifted items) divided by the total number of orders.

üìå Formula in the Data Warehouse:

sql
UPT = SUM(shopify__order__quantity_gift_excluded) / SUM(orders)

AOV (Average Order Value)

Represents the total sales revenue divided by the total number of orders.

üìå Formula in the Data Warehouse:

sql
AOV = SUM(Total Sales) / SUM(Orders)
